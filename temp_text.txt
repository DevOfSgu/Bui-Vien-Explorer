LU Ồ NG TR Ả I NGHI Ệ M CHI TI Ế T (CORE USER JOURNEY) H ệ th ố ng đượ c thi ế t k ế theo m ộ t vòng l ặ p s ử d ụ ng li ề n m ạ ch, bao g ồ m 4 giai đ o ạ n: Giai đ o ạ n 1: Kh ở i t ạ o thông minh (Smart Initialization) M ở App: App kh ở i t ạ o SessionId (phiên ẩ n danh). N ạ p AppSettings (ngôn ng ữ , giao di ệ n sáng/t ố i) t ừ SQLite. Data Check &amp; Background Sync: Ki ể m tra d ữ li ệ u Local. N ế u có m ạ ng, kích ho ạ t trigger t ả i ng ầ m (background sync) c ậ p nh ậ t Routes, Zones, ShopHours. Ng ườ i dùng l ướ t xem thông tin t ổ ng quan, ch ọ n m ộ t Tour c ụ th ể và b ấ m "Start Tour" để vào b ả n đồ chi ti ế t. Giai đ o ạ n 3: Đ i ề u h ướ ng t ạ i ch ỗ (Tour Map Execution) Map Rendering: Load toàn b ộ các đ i ể m d ừ ng (Zones) thu ộ c Tour lên b ả n đồ . Core GPS Logic: Ứ ng d ụ ng liên t ụ c l ấ y t ọ a độ user thông qua c ơ ch ế Stream k ế t h ợ p Polling fallback ( đả m b ả o không b ị r ớ t tín hi ệ u). THÔNG S Ố K Ỹ THU Ậ T LÕI (TECHNICAL ARCHITECTURE) 5.1. Thu ậ t toán X ử lý V ị trí (Advanced Location Engine) Để gi ả i quy ế t bài toán "nh ả y sai đ i ể m" ở môi tr ườ ng đ ô th ị h ẹ p nh ư Bùi Vi ệ n, h ệ th ố ng Mobile ph ả i tri ể n khai t ổ h ợ p thu ậ t toán sau: C ơ ch ế Stream + Polling Fallback: Ư u tiên dùng Event Stream (LocationChanged) c ủ a OS. N ế u quá 10s không có event (do vào h ẻ m khu ấ t), t ự độ ng kích ho ạ t Polling (h ỏ i OS v ị trí cu ố i cùng) để không b ị đ óng b ă ng UI. L ọ c nhi ễ u GPS (Noise Filtering): Lo ạ i b ỏ các đ i ể m t ọ a độ có độ chính xác (Accuracy) &gt; 50 mét. M ẫ u xác nh ậ n (Confirmation Pattern) &amp; Debounce: User ph ả i ở trong vùng bán kính (Geofence) liên t ụ c trong X giây (ví d ụ 3s) thì h ệ th ố ng m ớ i xác nh ậ n là " Đ ã vào Zone". Đ i ngang qua quá nhanh s ẽ b ị b ỏ qua. Hysteresis ( Độ tr ễ ng ưỡ ng): Tránh hi ệ n t ượ ng "Ping-Pong" (nh ả y qua l ạ i gi ữ a 2 POI sát nhau). Bán kính kích ho ạ t (Enter) là 15m, nh ư ng bán kính thoát (Exit) ph ả i là 20m. N ế u user đứ ng ở ranh gi ớ i 15m, vi ệ c xê d ị ch 1-2m do nhi ễ u GPS s ẽ không làm m ấ t trigger. 5.2. Ki ế n trúc Offline-First &amp; Đồ ng b ộ d ữ li ệ u Source of Truth (Ngu ồ n chân lý trên Mobile): SQLite (LocalZone, LocalRoute, LocalFavorite). Toàn b ộ UI đọ c tr ự c ti ế p t ừ SQLite, tuy ệ t đố i không g ọ i API để render giao di ệ n. C ơ ch ế Đồ ng b ộ (Sync Worker): Hành độ ng thêm/xóa Favorite: L ư u ngay vào Local -&gt; Push lên Queue -&gt; Có m ạ ng thì API Call lên Server -&gt; Nh ậ n 200 OK -&gt; Đ ánh d ấ u Synced. Hành độ ng t ả i Data: Fetch API -&gt; Replace SQLite -&gt; Update UI b ằ ng Event/Binding. 6. DANH SÁCH TÍNH N Ă NG (FEATURE SCOPE) 6.1.
